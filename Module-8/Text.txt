1. Explain the difference between local storage options (shared_preferences, SQLite, Hive).
->In Flutter apps, we need local storage to save data inside the device. There are different ways to do this depending on how much and what type of data we want to store.

~ shared_preferences

shared_preferences is the easiest way to store small data in Flutter.
It stores data in key-value format
Works like a simple dictionary 
Data stays saved even after closing the app

~ SQLite

SQLite is a full database system used inside mobile apps.

Data is stored in tables
Uses SQL queries like SELECT, INSERT, UPDATE, DELETE
Can handle large and complex data

~ Hive

Hive is a lightweight and very fast database for Flutter.

Stores data in key-value format
No SQL needed
Works directly with Dart objects
Very fast compared to SQLite

2. Describe CRUD operations and how they are implemented in SQLite or Hive
->CRUD means the four basic operations we do with data in any app.

~ C = Create

It means adding new data.
Example: Adding a new task in a to-do app

~ R = Read

It means viewing or getting data.
Example: Showing all tasks in the app

~ U = Update

It means changing existing data.
Example: Editing a task name

~ D = Delete

It means removing data.
Example: Deleting a completed task

3. Explain the advantages and use cases for shared_preferences.
->shared_preferences is very useful in Flutter for simple storage.

No setup of database needed
Fast and lightweight
Saves data even after closing the app

Saving login state 
Saving theme
